## Supabase Setup

Create a `songs` table with these columns:

```sql
create table public.songs (
  id bigint generated by default as identity primary key,
  title text not null,
  artist text not null,
  youtube_id text not null,
  rhythm_map jsonb not null default '[]'::jsonb
);
```

Create a `leaderboard_entries` table with these columns:

```sql
create table public.leaderboard_entries (
  id bigint generated by default as identity primary key,
  singer_name text not null,
  score integer not null check (score >= 0),
  song_title text not null default 'Karaoke Song',
  created_at timestamptz not null default now()
);
```

Example seed row:

```sql
insert into public.songs (title, artist, youtube_id, rhythm_map)
values (
  'LIFETIME - Reimagined',
  'BEN&BEN',
  'BhSZGUXeY6Q',
  '[
    {"start": 15.0, "end": 45.0, "label": "Verse 1"},
    {"start": 60.0, "end": 90.0, "label": "Chorus 1"}
  ]'::jsonb
);
```

Set these environment variables in Vercel or locally:

```bash
YOUTUBE_API_KEY=...
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_ANON_KEY=...
SUPABASE_SONGS_TABLE=songs
SUPABASE_LEADERBOARD_TABLE=leaderboard_entries
MOBILE_QUEUE_URL=https://your-mobile-queue-app.example.com
```

Notes:

- If `SUPABASE_URL` and a Supabase key are present, the app reads songs from Supabase.
- The leaderboard also reads from and writes to Supabase when that configuration is present.
- If they are absent, the app falls back to SQLite.
- `SUPABASE_SERVICE_ROLE_KEY` can be used instead of `SUPABASE_ANON_KEY` if you want server-side access that bypasses RLS.
- If you use the anon key, allow `select` on `public.songs` through your RLS policy.
- If you use the anon key for leaderboard features, allow `select`, `insert`, and `delete` on `public.leaderboard_entries`.
- Use the project API URL from `Project Settings -> API`, not the dashboard URL.
- Do not commit real Supabase keys into `.env.example` or your repo.
